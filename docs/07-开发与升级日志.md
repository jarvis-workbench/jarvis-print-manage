# 开发与升级日志

## 2026-03-28（Win7/Win10/Win11 兼容层加固）
- 主进程 PowerShell 执行器新增兼容层，针对旧版 PowerShell/Win7 补齐缺失能力：
  - ConvertTo-Json 兼容实现（PS2 无原生命令时降级到 JavaScriptSerializer）。
  - Get-CimInstance 兼容实现（降级到 Get-WmiObject）。
  - 打印机相关 cmdlet 兼容实现（Get/Add/Remove-Printer*、Get-PnpDevice 等缺失命令降级）。
- 执行器编码设置从 ::new(...) 改为 New-Object，避免旧版 PowerShell 语法不兼容。
- 依赖基线切换到 electron@22.3.27（Win7 支持分支）。

## 2026-03-28（备份 sidecar 元数据文件停用）
- 备份流程不再生成 *.pdrv.zip.driver-backup.json sidecar 文件。
- 备份索引统一以 driver-index.json 为唯一主数据来源，避免重复元数据文件并降低维护成本。
- 已移除对历史 sidecar 文件的读取逻辑。

## 2026-03-28（规划目录重构与编号治理）
- `docs/planning/` 重构为“按规划分文件夹”的结构：
  - `00001-printer-backup-archive/`
  - `00002-lan-network-sharing/`
  - `governance/`（治理文档）
- 两个规划目录均改为“目录内从 `00001` 连续编号”，不再使用根目录混排。
- 全量修正规划文档互相引用路径，避免迁移后链接失效。
- 新增关联规划协同优化要求：任一规划关键变更必须同步检查并回填关联规划。

## 2026-03-28（脚本外置规则落地 + 虚拟打印机配置化）
- `printer-state` worker 内联 PowerShell 已外置到 `electron/config/script/ps/printer-runtime-state.scps1`，worker 改为通过脚本加载器执行。
- 主进程压缩/解压 PowerShell 已外置到：
  - `printer-archive-create.scps1`
  - `printer-archive-extract.scps1`
- `sign.json` 已重生成，新增脚本纳入签名校验链路。
- 虚拟打印机判定规则从 `main.mjs` 硬编码迁移到 `electron/resource/config/virtual-printer.json`，主进程改为动态读取配置。

## 2026-03-28（打印机驱动备份压缩包化落地）
- 主进程索引标准化新增压缩包字段：`archiveFileName/archiveRelativePath/archiveSha256/archiveSize/archiveFormat/extractPolicy`。
- 备份流程升级为“目录备份后自动产出 `.pdrv.zip`”，并将压缩包哈希与大小写入 `driver-backup.json` 和 `driver-index.json`。
- 安装流程升级为“优先压缩包模式”：安装前校验压缩包存在与 SHA-256，自动解压到 `%TEMP%\\EleDrive\\extract\\<taskId>` 后继续安装链路。
- 安装完成自动清理解压临时目录；解压失败时抛出标准错误码（`ARCHIVE_EXTRACT_FAILED`）并保留上下文信息。
- 新增压缩包异常错误码：`ARCHIVE_NOT_FOUND`、`ARCHIVE_HASH_MISMATCH`、`ARCHIVE_EXTRACT_FAILED`。
- 前端类型契约同步扩展 `DriverBackupResult/DriverIndexEntry`，与主进程 `archive*` 返回字段保持一致。
- 备份完成后自动删除用于打包的临时驱动目录，避免目录与压缩包并存。
- 索引安装链路改为仅依赖压缩包字段，不再回退目录模式。
- 备份元数据改为压缩包同目录 sidecar 文件（`*.pdrv.zip.driver-backup.json`）用于索引重建（该方案后续已停用，见上方“备份 sidecar 元数据文件停用”条目）。

## 2026-03-28（命名规范收敛 + 规划制度落盘）

  - `printer-list-installed.scps1`
  - `printer-install-from-backup.scps1`
  - `printer-uninstall.scps1`
  - 其余脚本同步改为 `printer-*` 前缀。
- 主进程脚本调用标识同步调整，`sign.json` 重新生成。
- 新增规划目录 `docs/planning/` 并落盘两份规划文档：
  - `docs/planning/00001-printer-backup-archive/00001-主规划.md`
  - `docs/planning/00002-lan-network-sharing/00001-主规划.md`
- 新增制度文档《10-命名规范与规划制度》，固化“记住即落盘”“规划先文档后编码”的协作约束。

## 2026-03-28（稳定标识匹配 + PowerShell兼容性增强）
- 备份索引与运行态合并逻辑升级：优先使用 `PNPDeviceID / HardwareID / USB(VID:PID+Serial)` 匹配，不再仅依赖打印机名称。
- 备份元数据与索引新增稳定标识字段，解决“系统改名后，已还原打印机被误判为待还原”问题。
- 打印机安装入口改为优先使用 `backupPrinterName` 作为索引查找键，避免显示名变化导致查找失败。
- 新增 `electron/powershell.mjs` 统一执行层：支持绝对路径候选、`-NonInteractive`、超时控制、错误收敛。
- 主进程与打印机状态 worker 全部切换到统一 PowerShell 执行层，降低不同 Windows 环境下的兼容性风险。
- 更新脚本签名 `sign.json`，确保脚本完整性校验可通过。
- 将 worker 目录规范化：`printer-state.mjs` 放入 `electron/worker/`，为后续多 worker 扩展预留结构。
- PowerShell 模板后缀统一由 `*.ps1s` 改为 `*.scps1`，并同步调整脚本加载器与签名清单。
