## 1. 目标
- 让“重试/降级/回滚”可被稳定复现，不依赖人工随机制造故障。
- 提供统一注入开关，覆盖网络、协议、文件、安装四类异常。

## 2. 注入开关建议
- 配置文件：`<userData>/lan/fault-injection.json`
- 首版字段：
```json
{
  "enabled": false,
  "rules": {
    "dropDiscoveryPackets": 0,
    "transferFailAfterPercent": -1,
    "forceHashMismatch": false,
    "forcePairCodeInvalid": false,
    "forceInstallFail": false,
    "delayMs": 0
  }
}
```

规则说明：
- `dropDiscoveryPackets`：0-100，按概率丢弃发现包。
- `transferFailAfterPercent`：0-100，在指定进度触发传输失败，`-1` 表示关闭。
- `forceHashMismatch`：强制哈希不匹配。
- `forcePairCodeInvalid`：强制配对码校验失败。
- `forceInstallFail`：强制安装阶段失败。
- `delayMs`：人为注入延迟，验证超时与重试。

## 3. 场景矩阵（必须覆盖）
| 编号 | 注入规则 | 预期错误码 | 预期状态 | 额外校验 |
| --- | --- | --- | --- | --- |
| FI-01 | `dropDiscoveryPackets=100` | `LAN_DISCOVERY_TIMEOUT` | `FAILED` | 无无限重试 |
| FI-02 | `transferFailAfterPercent=40` | `LAN_TRANSFER_FAILED` | `RETRY_WAITING -> DONE/FAILED` | 重试次数<=3 |
| FI-03 | `forceHashMismatch=true` | `LAN_HASH_MISMATCH` | `FAILED` | 不进入自动重试 |
| FI-04 | `forcePairCodeInvalid=true` | `LAN_PAIR_CODE_INVALID` | `FAILED` | 配对流程终止 |
| FI-05 | `forceInstallFail=true` | `LAN_INSTALL_FAILED` | `FAILED` | 提示手工向导降级 |
| FI-06 | `delayMs=15000` | `LAN_NODE_UNREACHABLE` 或超时码 | `RETRY_WAITING` | 退避时间生效 |

## 4. 执行方式建议
- 本地调试：在 `dev` 模式允许热更新注入配置。
- CI/自动化：通过环境变量开启：
  - `ELEDRIVE_LAN_FAULT_INJECTION=1`
- 结果输出：每次执行产出 `fault-injection-report.json`。

## 6. 放行门槛
- FI-01 ~ FI-06 全通过，且无“任务卡死”现象。
- 任一场景失败，发布 Gate 直接标记“不通过”。
