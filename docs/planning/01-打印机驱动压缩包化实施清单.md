# 实施清单：打印机驱动压缩包化（P1）

更新时间：2026-03-28
状态：已确认（可进入编码）
关联规划：`docs/planning/打印机驱动备份压缩包化规划.md`

## 1. 实施目标（可交付定义）
- 备份输出从“目录”升级为“单文件压缩包 + 可校验索引”。
- 安装流程支持“自动解压 + 安装 + 清理临时目录”。
- 保持旧索引/旧目录备份可继续安装，不影响已落盘数据。

## 2. 字段变更清单

### 2.1 `driver-index.json`（新增字段）
- `archiveFileName: string`
- `archiveRelativePath: string`
- `archiveSha256: string`
- `archiveSize: number`
- `archiveFormat: 'pdrv.zip'`
- `extractPolicy: 'cleanup-on-success' | 'keep-on-fail'`

### 2.2 `driver-backup.json`（新增字段）
- 与索引字段保持同名，至少新增：
  - `archiveFileName`
  - `archiveRelativePath`
  - `archiveSha256`
  - `archiveSize`
  - `archiveFormat`

### 2.3 前端类型 `src/env.d.ts`（新增字段）
- `interface DriverIndexEntry` 同步新增：
  - `archiveFileName: string`
  - `archiveRelativePath: string`
  - `archiveSha256: string`
  - `archiveSize: number`
  - `archiveFormat: string`

## 3. 主进程改造点（按函数名）

### 3.1 `electron/main.mjs::normalizeIndex`
- 处理新增压缩字段默认值，确保旧索引读入不报错。

### 3.2 `electron/main.mjs::backupPrinterDriver`
- 在 `printer-backup-driver.scps1` 产出的目录基础上新增打包流程：
  - 生成 `<backupSubDir>.pdrv.zip`
  - 计算 SHA-256 与大小
  - 更新 `driver-backup.json` 与 `driver-index.json` 新字段

### 3.3 `electron/main.mjs::installPrinterFromBackup`
- 优先走“压缩包路径”：
  - 校验压缩包存在、哈希一致
  - 解压到 `%TEMP%/EleDrive/extract/<taskId>`
  - 用解压目录代替 `backupPath` 走现有安装脚本
- 若无压缩字段，则降级到旧目录路径（兼容模式）。

### 3.4 新增工具函数（建议）
- `createBackupArchive(backupPath): Promise<{ fileName, relativePath, sha256, size }>`
- `extractBackupArchive(archivePath, taskId): Promise<{ extractDir }>`
- `safeCleanupExtractDir(extractDir): Promise<void>`

## 4. IPC/API 变更清单（最小变更）
- 现有通道不新增，保持：
  - `printers:backup-driver`
  - `printers:install`
  - `drivers:index:get`
- 返回值扩展：
  - `backupPrinterDriver` 返回体增加 `archive*` 字段。
  - `drivers:index:get` 的 `entries[]` 增加 `archive*` 字段。
- 错误码建议（字符串常量，便于前端提示）：
  - `ARCHIVE_NOT_FOUND`
  - `ARCHIVE_HASH_MISMATCH`
  - `ARCHIVE_EXTRACT_FAILED`

## 5. 验收与回归矩阵

### 5.1 功能主路径
- 备份任意已安装打印机后生成 `.pdrv.zip`。
- 索引 `entries` 可读到 `archive*` 字段。
- 从索引触发安装可完成全流程。

### 5.2 兼容路径
- 旧索引（无 `archive*`）仍可安装。
- 索引重建后不丢原有身份字段（`pnpDeviceId/hardwareIds/usbVidPid/deviceSerial`）。

### 5.3 异常路径
- 手动删压缩包后安装应失败并提示 `ARCHIVE_NOT_FOUND`。
- 篡改压缩包后安装应失败并提示 `ARCHIVE_HASH_MISMATCH`。
- 解压失败时保留临时目录并返回路径。

## 6. 回滚策略（执行级）
- `settings` 增加临时开关（建议：`backupArchiveEnabled`，默认 `true`）。
- 当压缩流程异常时，可快速切回目录备份模式。
- 回滚不删除新增字段，保证前后版本索引可互读。

## 7. 开始编码前的最终确认
- 是否固定扩展名：`.pdrv.zip`（默认固定）。
- 是否固定哈希算法：`SHA-256`（默认固定）。
- 解压失败目录保留策略：默认保留到下次应用启动再清理。
