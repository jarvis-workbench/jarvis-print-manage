# 模块拆分草案：局域网共享（P2 准备）

更新时间：2026-03-28  
状态：草案（待 P1 完成后评审）  
关联前置清单：`docs/planning/02-局域网共享实施前置清单.md`

## 1. 目标
- 将“局域网共享”从方向规划细化为模块级拆分，降低 P2 启动成本。
- 提前冻结首版职责边界，避免发现层和传输层在实现时耦合。

## 2. 进程职责图（首版）
| 层 | 模块建议名 | 职责 | 明确不做 |
| --- | --- | --- | --- |
| main | `lan-discovery.mjs` | 节点发现、心跳、节点缓存 | 不处理文件传输 |
| main | `lan-service.mjs` | HTTP 接口、任务编排、白名单校验 | 不做复杂并发调度 |
| worker | `lan-transfer-worker.mjs` | 文件发送/下载、进度回传、重试 | 不做跨任务优先级抢占 |
| renderer | `lan-store.ts` | 节点列表、offer 列表、任务状态渲染 | 不直接访问文件系统 |

## 3. 端口与生命周期设计（首版）
- 固定配置项：
  - `lanEnabled: boolean`
  - `lanServicePort: number`（默认建议 `24821`）
  - `lanDiscoveryPort: number`（默认建议 `24822`）
- 生命周期约束：
  - 开关关闭：停止广播、停止监听、清理临时任务。
  - 开关开启：按“发现服务 -> HTTP 服务 -> UI 订阅”顺序启动。
  - 端口冲突：自动 +1 探测最多 10 次，失败则回退关闭态并给出错误码。

## 4. 任务状态机（discover -> offer -> transfer -> install）
| 状态 | 进入条件 | 退出条件 | 失败码建议 |
| --- | --- | --- | --- |
| `DISCOVERING` | 开启局域网开关 | 节点在线并握手成功 | `LAN_DISCOVERY_TIMEOUT` |
| `OFFER_READY` | 拉取远端能力成功 | 用户触发安装请求 | `LAN_OFFER_FETCH_FAILED` |
| `TRANSFERRING` | 远端接受请求并开始发送 | 本地文件校验通过 | `LAN_TRANSFER_FAILED` |
| `INSTALLING` | 压缩包可读且哈希正确 | 安装流程结束 | `LAN_INSTALL_FAILED` |
| `DONE` | 安装成功 | 任务归档 | - |

## 5. 与 P1 的接口契约
- 输入契约：局域网层只传递 `archivePath + archiveSha256 + archiveFormat`，不关心目录备份细节。
- 输出契约：安装层返回统一任务结果：
  - `taskId`
  - `status`
  - `errorCode?`
  - `errorMessage?`
- 兼容约束：若远端返回旧目录索引，直接拒绝并提示“提供者需升级到压缩包版”。

## 6. 评审 Gate（P2 开发前）
- Gate-1：模块职责是否无交叉写文件。
- Gate-2：状态机是否覆盖断网、取消、重试三类异常。
- Gate-3：错误码是否可映射到 UI 可读文案。
