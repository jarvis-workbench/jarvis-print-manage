# 规划：局域网驱动传输与对标 electron-hiprint 能力

更新时间：2026-03-29
状态：进行中（M1 开发中）

## 1. 目标
- 在 00002 局域网通讯底座上实现“局域网驱动传输业务”。
- 在此业务规划内推进对标 `CcSimple/electron-hiprint` 的打印服务能力。

## 2. 业务范围
- 局域网驱动包传输、安装任务编排、业务级错误回调。
- 打印服务协议与任务模型（对标能力在本规划中推进）。
- 渲染输出链路（fabric / gl-matrix / html2canvas）。

## 3. 边界
- 00003 只消费 00002 的通用发现/鉴权/状态机能力。
- 00003 不重写 00002 的底座协议与白名单机制。

## 4. 分阶段
1. 驱动传输业务闭环。
2. Socket 打印服务协议落地。
3. render-jpeg/pdf/print 三路输出。
4. 任务日志与重打控制。

## 5. 验收
- 局域网驱动传输闭环通过。
- 对标能力按 00002~00005 子文档里程碑逐项通过。

## 6. 最新进展（2026-03-29）
- 已完成 M1/L1 骨架落地：
  - 主进程新增 `print:service:*` IPC 与服务生命周期接入。
  - 新增 `print-socket-service`、`print-task-orchestrator`、`print-render-task` 模块骨架。
  - Preload/Store/类型定义已接入 `PrintServiceState/PrintJob`。
- 当前边界：
  - `news/render-*` 仅提供错误回调占位，真实打印与渲染执行在 M2/M3 落地。
