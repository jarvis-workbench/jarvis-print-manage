## 1. 里程碑总览
- M0：协议与任务模型冻结
- M1：Socket 服务与打印机信息事件
- M2：四类打印闭环
- M3：render 三路输出闭环
- M4：日志与重打
- M5：局域网融合与联调 Gate

## 2. 里程碑明细

### M0（1 周）
- 产物：
  - Socket 协议 V1 冻结
  - 打印任务状态机冻结
  - 错误码表冻结
- 通过标准：
  - 评审清单通过

### M1（1-2 周）
- 产物：
  - 本地 Socket 服务启动/停止
  - `clientInfo/printerList/getClientInfo/refreshPrinterList`
- 通过标准：
  - Node demo 可连接并获取设备与打印机信息

### M2（2 周）
- 产物：
  - `news` 支持 `html/pdf/url_pdf/blob_pdf`
  - `success/error` 回调
- 通过标准：
  - 四类打印均有成功与失败回调路径

### M3（2 周）
- 产物：
  - `render-jpeg/render-pdf/render-print`
  - fabric + gl-matrix + html2canvas 首版落地
- 通过标准：
  - 同一模板三路输出一致性通过

### M4（1 周）
- 产物：
  - 打印任务日志查询
  - 全局重打开关 + 单任务重打控制
- 通过标准：
  - 可回放最近 100 条任务并过滤失败原因

### M5（2 周）
- 产物：
  - 与 00002 局域网发现/配对/传输底座融合
  - 联调 Gate + 回滚演练 + 异常注入全通过
- 通过标准：
  - 端到端场景：远端驱动安装 + 本地 Socket 打印均稳定

## 3. 风险看板
- R1：渲染一致性偏差（高）
- R2：跨平台打印行为差异（高）
- R3：Socket 大 payload 内存压力（中）
- R4：协议兼容漂移（中）
