# 对标清单：electron-hiprint 能力映射与差距（V1）

更新时间：2026-03-28
状态：已确认（用于排期）
关联主规划：`docs/planning/00003-lan-driver-transfer-and-hiprint/00001-主规划.md`

## 1. 目标
- 明确 `ele-drive` 当前能力与 `electron-hiprint` 对标能力的差距。
- 给出分级优先级（P0/P1/P2），作为里程碑排期输入。

## 2. 对标矩阵

| 能力域 | 对标能力 | 当前状态 | 差距等级 | 备注 |
| --- | --- | --- | --- | --- |
| 通信 | 本地 Socket 服务监听与事件 | 未实现 | P0 | 需新建服务层 |
| 通信 | `token` 鉴权 | 未实现 | P0 | 与 00002 配对策略联动 |
| 通信 | `clientInfo/printerList` 推送 | 部分具备（本地 UI 有数据） | P0 | 需开放到 Socket 协议 |
| 打印 | HTML 静默打印 | 未实现 | P0 | 需任务模型与执行器 |
| 打印 | PDF 打印 | 未实现 | P0 | 需平台适配执行器 |
| 打印 | URL-PDF 打印 | 未实现 | P1 | 依赖下载与缓存策略 |
| 打印 | Blob-PDF 打印 | 未实现 | P1 | 依赖二进制 payload 规范 |

| 生态 | 中转服务接入 | 未实现 | P2 | 二阶段扩展 |

## 3. 优先级定义
- P0：构成“可用打印客户端”的最小闭环，不可延后。
- P1：提升可运维性、可用性和业务接入稳定性。
- P2：生态扩展与跨网络增强能力。

## 4. 执行建议
1. 先完成 P0：Socket 协议骨架 + 四类打印 + 三类渲染。
2. 再完成 P1：日志、重打、错误追踪、设置项完善。
3. 最后完成 P2：URL Scheme、中转服务、跨网段增强。

## 5. 验收口径
- P0 完成时，必须能通过 Node 客户端最小 Demo 完成全链路调用。
- P1 完成时，必须能回放至少 100 条打印任务并支持过滤排障。
- P2 完成时，必须新增兼容矩阵并通过跨版本联调。
