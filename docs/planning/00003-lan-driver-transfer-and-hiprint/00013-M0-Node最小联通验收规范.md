# M0 验收：Node 客户端最小联通脚本规范

更新时间：2026-03-28
状态：冻结（M1 验收必跑）
关联文档：
- `00005-实施路线图与里程碑.md`
- `00010-联调与发布前检查清单.md`

## 1. 目标
- 提供最小 Node 客户端脚本规范，用于验证 M1 基础联通。

## 2. 最小验证项
1. 建立连接（`socket.io-client`）
2. 接收 `clientInfo`
3. 接收 `printerList`
4. 主动发 `getClientInfo`
5. 主动发 `refreshPrinterList`

## 3. 通过标准
- 5 项全部通过，且错误回调结构化返回。
- 任一失败必须输出：
  - `event`
  - `timestamp`
  - `errorCode`
  - `message`

## 4. 脚本约束
- 使用 `transports=['websocket']`。
- 连接地址默认 `http://127.0.0.1:17521`。
- 连接超时默认 `5000ms`。

## 5. 扩展验证（M2）
- 在最小脚本基础上追加 `news` 与 `success/error` 验证。
