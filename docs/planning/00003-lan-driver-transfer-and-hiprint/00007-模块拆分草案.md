# 模块拆分草案：00003 业务能力

更新时间：2026-03-28
状态：草案（可进入评审）

## 1. 进程职责

| 层 | 模块建议名 | 职责 | 不做 |
| --- | --- | --- | --- |
| main | `print-socket-service.mjs` | Socket 服务、鉴权、事件路由 | 不处理模板渲染细节 |
| main | `print-task-orchestrator.mjs` | 打印任务编排、状态流转、日志写入 | 不做 UI 渲染 |
| worker | `print-render-task.mjs` | `render-*` 任务执行 | 不做网络发现 |
| renderer | `print-service-store.js` | 状态展示、任务筛选、重打入口 | 不直接访问文件系统 |
| renderer | `print-service-view.vue` | 客户端状态、任务面板 | 不写业务协议 |

## 2. 与 00002 边界
- 00003 通过 00002 暴露的节点与鉴权能力拿到可访问通道。
- 00003 不直接操作 00002 的底层发现广播实现。

## 3. 任务流
- `news` -> `QUEUED` -> `RUNNING` -> `DONE/FAILED`
- `render-*` -> `QUEUED` -> `RUNNING` -> `DONE/FAILED`
- 失败进入统一错误码与日志格式。
