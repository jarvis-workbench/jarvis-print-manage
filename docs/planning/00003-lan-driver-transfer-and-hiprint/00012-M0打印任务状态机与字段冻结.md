## 1. 状态定义
- `QUEUED`
- `RUNNING`
- `DONE`
- `FAILED`
- `CANCELED`

## 2. 流转规则
- `QUEUED -> RUNNING -> DONE`
- `QUEUED -> RUNNING -> FAILED`
- `QUEUED -> CANCELED`
- `RUNNING -> CANCELED`

约束：
- `DONE/FAILED/CANCELED` 为终态，不允许再迁移。

## 3. 最小字段冻结（`PrintJob`）
```json
{
  "taskId": "print-task-uuid",
  "templateId": "biz-001",
  "type": "html|pdf|url_pdf|blob_pdf|render-jpeg|render-pdf|render-print",
  "status": "QUEUED|RUNNING|DONE|FAILED|CANCELED",
  "printer": "Printer Name",
  "errorCode": "",
  "errorMessage": "",
  "createdAt": "2026-03-28T12:00:00.000Z",
  "updatedAt": "2026-03-28T12:00:01.000Z"
}
```

## 4. 事件回填约束
- 每次状态变化必须更新 `updatedAt`。
- 进入 `FAILED` 时必须填 `errorCode`。
- 进入 `DONE` 时必须清空 `errorCode/errorMessage`。

## 5. 任务追踪约束
- Socket 回调、IPC 查询、日志记录必须共用同一个 `taskId`。
