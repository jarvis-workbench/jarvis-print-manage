# 规划评审检查清单（通用）

更新时间：2026-03-28
用途：每个规划进入“可编码”前，必须逐条评审并记录结论。

## 1. 目标与边界
- [ ] 目标是“可交付结果”，不是方向描述。
- [ ] 明确了“不做什么”。
- [ ] 明确了首版 MVP 边界。

## 2. 数据与接口
- [ ] 字段变更写清楚了“新增/修改/兼容默认值”。
- [ ] IPC/API 变更写清楚了“通道名/参数/返回值/错误码”。
- [ ] 旧数据兼容路径已定义。

## 3. 实施与回滚
- [ ] 有按阶段可落地的实施节点。
- [ ] 每阶段都有可判定完成标准。
- [ ] 出错时有明确回滚路径（功能、数据、运维）。

## 4. 验证与发布
- [ ] 至少覆盖主路径、兼容路径、异常路径三类用例。
- [ ] 关键日志点已定义（能定位失败段）。
- [ ] 明确上线 gate（满足哪些条件才可下一阶段）。

## 5. 依赖与阻塞
- [ ] 前置依赖明确且可验证。
- [ ] 待确认问题可落为“是/否”决策。
- [ ] 阻塞项有负责人或下一步动作。

## 6. 评审结论模板
- 评审日期：`YYYY-MM-DD`
- 评审结论：`通过 / 有条件通过 / 不通过`
- 阻塞项：
  - `...`
- 放行条件：
  - `...`
