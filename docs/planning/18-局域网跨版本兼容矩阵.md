# 设计：局域网跨版本兼容矩阵（P2/P3）

更新时间：2026-03-28  
状态：已确认（发布与联调基线）

- `docs/planning/10-局域网传输失败重试与降级策略.md`

## 1. 目标
- 明确不同客户端版本之间的可发现、可配对、可安装边界，避免灰度期“能发现但不能装”的隐性故障。
- 将兼容判定转成联调和 CI 的可执行矩阵。

## 2. 版本分层约定
- `AppVersion`：应用版本（语义化版本，例如 `1.6.0`）。
- `LanProtocolVersion`：局域网协议版本（整数，自增）。
- `DriverArchiveVersion`：驱动压缩包格式版本（整数，自增）。

首版冻结：
- `LanProtocolVersion = 1`
- `DriverArchiveVersion = 1`

## 3. 兼容策略
- 同主协议兼容：`LanProtocolVersion` 相同即可互通。
- 高版本访问低版本：允许发现与配对；若能力字段缺失，返回明确降级错误。
- 低版本访问高版本：允许发现；若协议字段超出已知集合，返回 `LAN_PROTOCOL_MISMATCH`。
- 压缩包格式不兼容时：禁止安装并给出手工安装引导。

## 4. 跨版本兼容矩阵

| 发起端 App | 目标端 App | 协议组合 | 发现 | 配对 | 拉取 offers | 安装 | 预期结果 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 1.6.x | 1.6.x | P1 -> P1 / A1 -> A1 | 是 | 是 | 是 | 是 | 全链路通过 |
| 1.7.x | 1.6.x | P1 -> P1 / A1 -> A1 | 是 | 是 | 是 | 是 | 向后兼容通过 |
| 1.6.x | 1.7.x | P1 -> P1 / A1 -> A1 | 是 | 是 | 是 | 是 | 向前兼容通过 |
| 1.8.x | 1.6.x | P2 -> P1 / A2 -> A1 | 是 | 条件是 | 否 | 否 | 返回协议不兼容并降级 |
| 1.6.x | 1.8.x | P1 -> P2 / A1 -> A2 | 是 | 否 | 否 | 否 | 返回协议不兼容并降级 |

注：
- `P1/P2` 代表 `LanProtocolVersion 1/2`。
- `A1/A2` 代表 `DriverArchiveVersion 1/2`。

## 5. 错误码映射
- 协议不兼容：`LAN_PROTOCOL_MISMATCH`
- 压缩包格式不兼容：`LAN_ARCHIVE_VERSION_UNSUPPORTED`
- 字段缺失导致不可安装：`LAN_OFFER_SCHEMA_INVALID`
- 仅可发现不可安装的降级态：`LAN_DEGRADED_DISCOVERY_ONLY`

## 6. 实施要求
- 节点发现返回必须包含：
  - `appVersion`
  - `lanProtocolVersion`
  - `driverArchiveVersion`
- `lan:list-offers` 响应中必须携带 `archiveFormatVersion`。
- `lan:request-install` 在进入 `TRANSFERRING` 前必须完成版本门禁校验。

## 7. 联调与CI用例映射
- 联调新增用例：
  - `CV-01` 同版本互通
  - `CV-02` 新版向后兼容
  - `CV-03` 旧版访问新版降级
  - `CV-04` 协议不兼容错误码校验
  - `CV-05` 压缩包版本不兼容错误码校验
- CI 映射：
  - `contract-tests` 执行 `CV-01~CV-05`
  - `fault-injection-tests` 附加 `CV-04/CV-05` 降级路径验证

## 8. 放行门槛
- `CV-01~CV-03` 通过率 100%。
- `CV-04~CV-05` 必须稳定返回预期错误码且保留可读提示。
- 任一版本组合出现“无错误码的静默失败”即阻断发布。
