# 规划：打印机驱动备份输出为压缩包

更新时间：2026-03-28
状态：评审中（未开始编码）

## 1. 目标
- 将当前“备份输出目录”改为“备份输出压缩包（单文件）”。
- 保持现有安装流程可用：从备份索引安装时自动解压并安装。
- 提升备份可迁移性与传输效率，为后续局域网传输复用。

## 2. 范围
- 包含：
  - 备份流程：目录打包为 `.zip`（建议扩展名 `*.pdrv.zip`）。
  - 索引结构扩展：记录压缩包路径、包内 INF 相对路径、校验信息。
  - 安装流程：安装前自动解压到临时目录并清理。
- 不包含：
  - 云端存储/断点续传（后续规划）。
  - 跨平台驱动包转换（仅 Windows 目标）。

## 3. 现状与问题
- 当前备份是目录结构，跨机传输需要整目录拷贝，用户操作成本高。
- 目录模式文件碎片多，后续局域网共享时传输和校验复杂。

## 4. 方案设计
## 4.1 备份产物
- 每次备份输出：
  - `xxx-时间戳.pdrv.zip`（驱动包）
  - 索引中记录：
    - `archiveFileName`
    - `archiveSha256`
    - `archiveSize`
    - `infRelativePath`
    - 现有标识字段（`pnpDeviceId`、`hardwareIds`、`usbVidPid` 等）

## 4.2 安装流程
- 安装入口接收索引项后：
  - 检查压缩包存在性 + 哈希。
  - 解压到临时目录（`%TEMP%\\EleDrive\\extract\\<taskId>`）。
  - 按原逻辑定位 INF 并调用安装脚本。
  - 安装完成后清理临时目录（失败时保留并给出路径用于排查）。

## 4.3 兼容策略
- 兼容旧目录备份：
  - 若索引项无 `archiveFileName`，沿用旧目录安装路径。
  - 提供“索引重建”时自动补全压缩字段（可选任务，非首期强制）。

## 5. 实施节点
1. 数据结构升级
2. 备份流程新增压缩与哈希
3. 安装流程新增解压与清理
4. UI 提示优化（显示“压缩包备份”状态）
5. 回归测试与兼容验证

## 6. 风险与应对
- 风险：压缩/解压失败导致安装中断
  - 应对：明确错误码与日志，保留临时目录。
- 风险：旧索引与新索引并存
  - 应对：双路径兼容 + 渐进迁移。
- 风险：大包耗时
  - 应对：后台任务 + 进度回调（第二阶段）。

## 7. 验收标准
- 备份后索引可定位到压缩包并可校验。
- 从压缩包恢复安装成功率与旧流程一致。
- 旧目录备份不回归，仍可安装。

## 8. 建议优先级
- 优先级：P1（为局域网驱动共享提供基础能力）。

## 9. 前置依赖
- 现有目录备份链路稳定可用（当前已满足）。
- 安装入口支持根据索引信息定位 INF 并执行安装（当前已满足）。
- 主进程具备临时目录读写权限与清理能力（需在实现阶段补校验）。

## 10. 待确认问题
- 压缩格式是否固定为 `*.pdrv.zip`（建议固定，避免多格式分支）。
- 哈希算法是否统一 `SHA-256`（建议统一，避免索引字段碎片化）。
- 解压失败时保留目录的默认时长（建议保留到下次应用启动再清理）。

## 11. MVP 边界（首版）
- 必做：
  - 新备份输出单文件压缩包 + 哈希写入索引。
  - 安装时自动解压并复用现有安装链路。
  - 兼容旧目录索引安装。
- 不做：
  - 分卷压缩、断点续传、增量备份。
  - 远程分发能力（由局域网规划承接）。

## 12. 回滚策略
- 代码开关回滚：当压缩备份异常时，可切回目录备份流程。
- 数据兼容回滚：保留旧索引解析路径，新字段缺失不报错。
- 运维回滚：失败任务输出错误码并保留临时目录路径，支持人工排查后重试。
