# 草案：局域网共享 IPC 与数据契约（P2/P3）

更新时间：2026-03-28  
状态：草案（待 P1 完成后评审）  
关联模块：`docs/planning/00002-lan-network-sharing/00003-模块拆分草案.md`

## 1. 目标
- 为 P2/P3 预定义 Electron IPC 边界，避免 renderer 直接依赖局域网实现细节。
- 约束“节点发现/能力展示/传输安装”三段数据结构，保证日志与状态可追踪。

## 2. 建议 IPC 通道（新增）
- `lan:get-state`：读取局域网总状态。
- `lan:set-enabled`：启停局域网能力。
- `lan:list-nodes`：读取在线节点。
- `lan:list-offers`：读取远端可安装项。
- `lan:request-install`：请求远端传输并本地安装。
- `lan:get-task`：读取任务状态。
- `lan:cancel-task`：取消任务。

## 4. IPC 入参与返回（首版）

### 4.1 `lan:set-enabled`
- 入参：`{ enabled: boolean }`
- 返回：`{ enabled: boolean startedAt?: string errorCode?: string }`

### 4.2 `lan:request-install`
- 入参：`{ nodeId: string offerId: string targetPrinterName?: string }`
- 返回：`{ taskId: string status: string }`

### 4.3 `lan:get-task`
- 入参：`{ taskId: string }`
- 返回：`LanTask`

## 5. 错误码建议
- 发现层：`LAN_DISCOVERY_TIMEOUT` / `LAN_NODE_UNREACHABLE`
- 协议层：`LAN_OFFER_FETCH_FAILED` / `LAN_PROTOCOL_MISMATCH`
- 传输层：`LAN_TRANSFER_FAILED` / `LAN_HASH_MISMATCH`
- 安装层：`LAN_INSTALL_FAILED` / `LAN_ARCH_MISMATCH`

## 6. 与 P1 的衔接约束
- 仅接受 `archiveFormat = pdrv.zip` 的 offer。
- 若远端无 `archiveSha256`，拒绝安装并返回 `LAN_PROTOCOL_MISMATCH`。
- `lan:request-install` 内部必须复用 P1 `printers:install` 安装链路，不重复造一套安装流程。

## 7. 评审通过条件
- renderer 仅依赖上述 IPC，不直连网络模块。
- 日志可按 `taskId` 串起发现/传输/安装三段。
- 取消任务在 `TRANSFERRING` 阶段可生效并释放临时文件。
