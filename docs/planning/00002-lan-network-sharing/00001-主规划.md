# 规划：局域网通讯底座（供后续网络业务复用）

更新时间：2026-03-28
状态：评审中（未开始编码）

## 1. 目标
- 构建统一、稳定、可复用的局域网通讯环境。
- 为后续网络业务提供通用能力：节点发现、配对鉴权、白名单、任务状态、传输通道。
- 首版仅解决“同网段内可靠通讯”，不承载具体业务协议细节。

## 2. 定位与边界
- 本规划是“底座层”，不是业务功能实现层。
- 本规划不直接定义 Socket 打印服务，不直接承载 electron-hiprint 对标能力。
- 后续业务（如驱动传输、打印服务）通过独立规划复用本底座。

## 3. 范围
- 包含：
  - 同网段 IPv4 节点发现与在线状态维护。
  - 节点配对鉴权、白名单、令牌与基础安全门禁。
  - 通用任务状态机与日志规范。
  - 错误码、兼容矩阵、回滚与 CI 基线。
- 不包含：
  - 具体业务 payload（打印、模板渲染、云中继等）。
  - 跨子网穿透、企业级权限体系。

## 4. 核心能力
- 发现层：UDP 广播 + 心跳，维护节点生命周期。
- 鉴权层：一次性配对码 + 白名单 + 限流。
- 通道层：通用 HTTP 接口与任务查询能力。
- 状态层：统一状态机、重试与降级策略。
- 可观测层：JSONL 任务日志、轮转与清理策略。

## 5. 设计原则
- 通讯底座与业务协议分离。
- 状态机优先，错误码统一。
- 能回滚、可诊断、不污染本地核心链路。

## 6. 分阶段实施
1. 第一阶段：发现与在线状态。
2. 第二阶段：配对鉴权与白名单。
3. 第三阶段：任务状态机与日志规范。
4. 第四阶段：失败重试、降级、回滚验证。
5. 第五阶段：兼容矩阵与 CI 自动化固化。

## 7. 验收标准
- 三台节点 24h 发现与在线状态稳定。
- 未配对节点无法访问受限接口。
- 任务状态可完整回放，异常链路可定位。
- 关闭局域网开关后，发现/监听/任务均可有序停止。
