# 策略草案：局域网传输失败重试与降级（P3）

更新时间：2026-03-28  
状态：草案（待 P1 完成后评审）  
关联文档：
- `docs/planning/00002-lan-network-sharing/00004-IPC与数据契约草案.md`

## 1. 目标
- 固化 P3 传输与安装失败时的重试边界，避免无限重试与状态失控。
- 定义“可重试/不可重试”分类，统一前端提示与日志语义。

## 2. 重试总规则
- 单任务最大重试次数：`3`。
- 重试退避：`2s -> 5s -> 10s`。
- 达到上限后进入 `FAILED`，需要用户手动重新发起。

## 3. 错误分类

### 3.1 可重试（自动）
- `LAN_NODE_UNREACHABLE`
- `LAN_TRANSFER_FAILED`
- `LAN_DISCOVERY_TIMEOUT`

### 3.2 不可重试（直接失败）
- `LAN_ARCH_MISMATCH`
- `LAN_PROTOCOL_MISMATCH`
- `LAN_HASH_MISMATCH`
- `LAN_PAIR_BLOCKED`

### 3.3 需用户决策
- `LAN_INSTALL_FAILED`
  - 提示用户：重试安装 / 改为手工向导安装。

## 4. 状态机补充
- `TRANSFERRING` 失败且可重试：进入 `RETRY_WAITING`，记录 `retryCount`。
- `RETRY_WAITING` 到时后回到 `TRANSFERRING`。
- 达上限后：`FAILED`，写入最终失败事件。

## 5. 日志事件补充
新增事件：
- `RETRY_SCHEDULED`
- `RETRY_STARTED`
- `RETRY_EXHAUSTED`

新增字段：
- `retryCount: number`
- `maxRetry: number`
- `nextRetryAt?: string`

## 6. 用户提示规范
- 自动重试中：`正在重试（第 X/Y 次）`。
- 重试耗尽：`安装失败，已达到最大重试次数，请稍后重试。`
- 不可重试：显示明确原因并给出下一步动作。

## 7. 降级策略
- 传输连续失败时，允许用户导出“远端压缩包下载链接”（仅同网段有效，5 分钟过期）。
- 自动链路失败后可降级到“手工安装向导 + 本地包路径”。

## 8. 验收点
- 可重试错误在 3 次内成功时任务最终为 `DONE`。
- 不可重试错误首次失败即 `FAILED`，无额外重试。
- `RETRY_*` 事件在 `lan-task-log.jsonl` 可完整回放。
