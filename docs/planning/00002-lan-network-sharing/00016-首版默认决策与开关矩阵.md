## 1. 目标
- 将当前“待确认问题”转成“默认可执行决策”，避免编码阶段因口径不一致阻塞。
- 约束所有偏离默认值的实现必须走评审，防止隐式改协议。

## 2. 首版默认决策（冻结）

| 决策项 | 默认值 | 生效范围 | 偏离策略 |
| --- | --- | --- | --- |
| 驱动压缩扩展名 | `.pdrv.zip` | P1 备份产物、P2/P3 传输载体 | 需新增 `ArchiveVersion` 并走兼容评审 |
| 哈希算法 | `SHA-256` | 备份索引、传输校验、安装前校验 | 禁止混用；若升级需双写过渡 |
| 节点鉴权 | 一次性配对码 + 白名单缓存 | P2/P3 节点配对与授权 | 若改为长期令牌，需新增吊销机制 |
| 首版传输协议 | HTTP（仅内网） | P3 文件传输 | 迁移 TLS 时保持错误码与回滚路径不变 |
| 发现范围 | 同网段 IPv4 广播 | P2 节点发现 | 跨网段能力不在首版范围 |
| 节点 ID 生成 | 首启生成 UUID 并持久化 | P2/P3 节点唯一标识 | 重置需明确“清空白名单”联动 |

## 3. 功能开关矩阵

| 开关键 | 默认值 | 控制范围 | 关闭时行为 |
| --- | --- | --- | --- |
| `feature.backup.archiveEnabled` | `true` | 压缩包备份（P1） | 回退目录备份路径 |
| `feature.lan.discoveryEnabled` | `false` | 局域网发现（P2） | 不监听、不广播 |
| `feature.lan.transferEnabled` | `false` | 局域网传输（P3） | 只展示只读能力，不触发安装 |
| `feature.lan.autoInstallEnabled` | `false` | 传输后自动安装 | 下载后停留人工确认 |

说明：
- 首次灰度建议仅开启 `backup.archiveEnabled`。
- `lan.discoveryEnabled` 与 `lan.transferEnabled` 不允许跳级开启（必须先发现后传输）。

## 4. 配置与契约落盘要求
- 配置文件中必须显式记录：
  - `lanProtocolVersion`
  - `driverArchiveVersion`
  - 四个功能开关键
- 节点握手返回必须包含：
  - `nodeId`
  - `appVersion`
  - `lanProtocolVersion`
  - `driverArchiveVersion`
- 任何接口新增字段必须同步更新：
  - `docs/planning/00002-lan-network-sharing/00004-IPC与数据契约草案.md`
  - `docs/planning/00002-lan-network-sharing/00015-跨版本兼容矩阵.md`

## 5. 放行条件（默认决策层）
- 开发放行：默认值全部在代码中有常量定义与单测覆盖。
- 联调放行：跨版本用例 `CV-01~CV-05` 全通过。
- 发布放行：功能开关能独立回滚且回滚后不影响本地安装链路。

## 6. 变更门禁
- 出现以下任一情况必须发起评审，不允许直接编码：
  - 修改压缩扩展名或哈希算法。
  - 新增非 HTTP 传输形态（如 TLS/mTLS）。
  - 允许跨网段发现或中继传输。
  - 调整 `nodeId` 生成/重置策略。
- 评审结论写入：
  - `docs/planning/governance/00002-规划评审检查清单.md` 的“评审结论模板”。

## 7. 回滚基线
- 发现异常时按优先级回滚：
1. 关闭 `feature.lan.transferEnabled`，保留发现只读。
2. 关闭 `feature.lan.discoveryEnabled`，完全退出局域网能力。
3. 关闭 `feature.backup.archiveEnabled`，恢复目录备份。
- 回滚后必须保留：
  - 错误码
  - 任务日志
  - 临时目录定位信息
