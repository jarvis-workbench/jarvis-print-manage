# 规划目录总览（docs/planning）

更新时间：2026-03-28

## 1. 目录规范（强制）
- 每个“独立规划”必须使用一个独立文件夹，命名规则：`00001-...`、`00002-...`。
- 编号从 `00001` 开始连续增长，不允许无编号规划。
- 同一规划的扩展文档必须放在该规划文件夹内，且在该文件夹内从 `00001-...` 连续编号。
- 不同规划之间不允许文档编号穿插；跨规划关联通过“关联规划”字段显式声明。

## 2. 当前结构
- `governance/`：规划治理与评审资产（非业务规划）。
- `00001-printer-backup-archive/`：打印机驱动备份压缩包化规划及扩展。
- `00002-lan-network-sharing/`：局域网通讯底座规划及扩展。
- `00003-lan-driver-transfer-and-hiprint/`：局域网驱动传输与对标能力规划及扩展。

## 3. 执行顺序（禁止穿插）
1. 先完成 `00001-printer-backup-archive/` 主路径闭环与验收。
2. 再完成 `00002-lan-network-sharing/` 通讯底座能力。
3. 最后进入 `00003-lan-driver-transfer-and-hiprint/` 业务能力实现。

## 4. 关联规划联动规则（强制）
- 每个规划主文档必须包含“关联规划与协同优化”章节。
- 当某规划新增/调整关键约束时，必须同步检查并更新关联规划中的对应条目（依赖、接口、验收、回滚）。
- 关联更新结果必须写入：
  - 该规划目录内扩展文档；
  - `governance/00001-规划迭代记录.md`。

## 5. 快速入口
- 主规划 A：`00001-printer-backup-archive/00001-主规划.md`
- 主规划 B：`00002-lan-network-sharing/00001-主规划.md`
- 主规划 C：`00003-lan-driver-transfer-and-hiprint/00001-主规划.md`
- 治理记录：`governance/00001-规划迭代记录.md`
- 评审清单：`governance/00002-规划评审检查清单.md`
