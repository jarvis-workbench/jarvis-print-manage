# 规划文档目录（docs/planning）

更新时间：2026-03-28

本目录用于落实“先规划、后评审、再编码”的协作流程。
所有规划必须可执行、可验收、可回滚，避免只有方向描述没有落地路径。

## 1. 文档状态定义
- `草案`：仅形成初版，未评审。
- `评审中`：已进入讨论，存在待确认问题。
- `已确认`：关键问题已确认，可进入开发排期。
- `实施中`：已开始编码或联调。
- `已完成`：功能落地并通过验收。
- `搁置`：因优先级/条件变化暂停。

## 2. 单个规划最小结构
每个规划文档至少包含：
- 目标与范围（包含“不包含”边界）
- 现状与问题
- 方案设计（接口/数据结构/流程）
- 分阶段实施节点
- 风险、回滚与兼容策略
- 验收标准
- 前置依赖与待确认问题

## 3. 目录维护规则
- 每个规划单独一个 `.md` 文件，使用语义化中文文件名。
- 规划状态变化时，必须同步更新文档头部“状态”和“更新时间”。
- 当规划进入 `已确认` 或 `实施中`，需在 `docs/07-开发与升级日志.md` 追加里程碑记录。
- 每轮自动化迭代后，将关键决策写入：
  - `docs/planning/00-规划迭代记录.md`
