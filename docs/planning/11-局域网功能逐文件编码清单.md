## 1. 目标
- 将 P2/P3 拆解到真实文件级任务，明确每一步输入、输出、完成判定。

## 2. 批次规划
- L1：基础设施与配置（main + preload + types）
- L2：节点发现与配对鉴权（main + store + settings）
- L3：能力展示与任务发起（main + view + store）
- L4：传输安装与重试降级（worker + main + view）
- L5：日志轮转与回归验收（main + docs）

## 3. 逐文件任务

### 3.1 `electron/main.mjs`
- 新增局域网 IPC：`lan:get-state`、`lan:set-enabled`、`lan:list-nodes`、`lan:list-offers`、`lan:request-install`、`lan:get-task`、`lan:cancel-task`、`lan:pair:*`。
- 新增白名单与日志读写入口（调用 `lan` 模块，不在 main 内堆逻辑）。
- 完成判定：IPC 接口可返回稳定结构，异常时返回标准错误码。

### 3.2 `electron/preload.mjs`
- 暴露 `window.eleDrive.lan*` API，参数与返回严格对齐 `06/09/10`。
- 完成判定：renderer 无需直接访问 Node/Electron API。

### 3.3 `src/env.d.ts`
- 增加类型：`LanNode`、`RemotePrinterOffer`、`LanTask`、`LanPairState`。
- 完成判定：`npm run typecheck` 无新增类型错误。

### 3.4 `src/stores/runtime.js`
- 新增局域网运行态：
  - `lanEnabled`
  - `lanNodes`
  - `lanOffers`
  - `lanTasks`
- 新增动作：`refreshLanNodes/refreshLanOffers/requestLanInstall/cancelLanTask`。
- 完成判定：页面刷新后可恢复状态，失败时有错误态。

### 3.5 `src/views/SettingsView.vue`
- 新增“局域网组网”开关区、配对管理入口、白名单管理入口。
- 完成判定：开关行为与 `lan:set-enabled` 一致，能显示错误状态。

### 3.6 `src/views/PrintersView.vue`
- 新增远端 offers 列表与“安装到本机”动作。
- 显示任务状态与重试提示（第 X/Y 次）。
- 完成判定：能从远端列表发起任务并跟踪状态变化。

### 3.7 `electron/worker/`（新增）
- 新增：`lan-transfer.mjs`（建议名）
- 职责：传输、进度上报、重试调度、降级输出。
- 完成判定：可重试错误进入 `RETRY_WAITING`，不可重试直接 `FAILED`。

### 3.8 `electron/config/script/ps/`（可选）
- 若需本地探测/诊断，新增最小脚本并更新 `sign.json`。
- 完成判定：脚本签名校验通过，调用链路可用。
