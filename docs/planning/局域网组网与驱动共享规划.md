# 规划：局域网组网、节点发现与驱动共享安装

更新时间：2026-03-28
状态：评审中（未开始编码）

## 1. 目标
- 在系统设置中提供“局域网组网”开关。
- 多台安装本软件的电脑在同一局域网内自动发现彼此并组成可通信网络。
- 在组网环境下，显示“他机有且本机无”的打印机/驱动清单，并支持一键安装到本机。

## 2. 核心场景
- A/B/C 三台电脑开启“局域网组网”。
- A 能看到 B/C 的可共享打印机（与 A 本机去重后）。
- A 点击安装某项后：
  - 若提供者未备份，则提供者先自动备份并打包。
  - 提供者传输驱动包给 A。
  - A 下载完成后自动安装，名称/端口信息复制提供者配置（无需再手工输入）。

## 3. 范围
- 包含：
  - 节点发现、握手认证、在线状态维护。
  - 打印机能力广播与去重视图。
  - 驱动包按需备份、传输、自动安装。
  - 架构区分 x64/x86，安装前匹配。
- 不包含：
  - 跨子网穿透、云中继（后续阶段）。
  - 域账号/企业级权限系统（后续阶段）。

## 4. 总体架构
## 4.1 节点发现层
- 建议：UDP 广播 + 周期心跳（本地网段）。
- 每节点发布：
  - `nodeId`
  - `machineName`
  - `appVersion`
  - `arch`（x64/x86）
  - `servicePort`

## 4.2 通信层
- 建议：HTTP(s) + WebSocket（本地服务，仅局域网监听）。
- 功能接口：
  - 查询节点打印机能力
  - 请求驱动包元数据
  - 拉取驱动包文件
  - 进度与结果回调

## 4.3 数据模型（建议）
- `LanNode`：节点信息、在线状态、能力摘要。
- `RemotePrinterOffer`：提供者打印机元数据（驱动名、版本、端口、架构、稳定标识）。
- `TransferTask`：传输任务（状态、进度、错误）。
- `InstallTask`：安装任务（状态、日志、结果）。

## 5. 关键策略
## 5.1 去重策略
- 优先使用稳定标识去重（`pnp/hardwareId/usbVidPid+serial`）。
- 无稳定标识时，降级为 `driverName + portHost + manufacturer`。

## 5.2 架构匹配
- 安装前强校验包架构与目标系统架构。
- 不匹配时给出明确提示，不进入安装阶段。

## 5.3 自动备份与传输
- 提供者接到请求时：
  - 若已有可用压缩包，直接发送。
  - 若无，先执行自动备份打包，再发送。

## 5.4 安装行为
- 安装者侧自动套用提供者配置：
  - 打印机名称
  - 端口/IP
  - 驱动参数
- 默认无需人工输入（异常场景可回退至向导）。

## 6. 安全与权限
- 最低要求：
  - 节点配对码/一次性授权确认。
  - 白名单节点缓存（可移除）。
  - 本地防火墙端口范围固定可控。
- 后续增强：
  - 传输签名校验、包哈希校验、可选 TLS。

## 7. 分阶段实施
1. 第一期：节点发现 + 在线列表（只读）
2. 第二期：远端打印机去重展示（只读）
3. 第三期：驱动包请求与传输（手动触发）
4. 第四期：传输完成自动安装（全自动链路）
5. 第五期：安全与可观测性完善（日志、重试、审计）

## 8. 验收标准
- 三台节点可稳定发现并展示在线状态。
- 本机可查看“他机有且本机无”的打印机列表。
- 请求安装后可自动完成：备份（若需要）-> 传输 -> 安装。
- 架构不匹配时明确失败且不污染本机状态。

## 9. 里程碑建议
- M1（1-2 周）：发现层 + UI 开关与节点列表。
- M2（1-2 周）：去重模型 + 远端清单。
- M3（2-3 周）：传输协议 + 自动安装闭环。

## 10. 前置依赖
- 已完成“打印机驱动备份压缩包化”首版闭环（当前未满足，阻塞）。
- 设置页具备局域网开关与状态展示区域（部分满足，需补交互细节）。
- 主进程具备本地服务端口管理能力（需评估现有 IPC/进程模型）。

## 11. 待确认问题
- 首版是否只支持同网段 IPv4 广播，不处理跨网段。
- 首版传输是否允许 HTTP 明文，仅依赖配对码 + 白名单。
- 节点标识 `nodeId` 的生成策略（建议：首次启动生成并持久化）。

## 12. MVP 边界（首版）
- 必做：
  - 节点发现、在线列表、远端打印机只读展示。
  - 手动触发单任务传输与安装。
  - 失败可追踪日志（节点/传输/安装三段）。
- 不做：
  - 多任务并发调度、断点续传、跨子网中继。
  - 企业级权限体系与组织级审计。

## 13. 回滚策略
- 功能开关回滚：关闭“局域网组网”后停止发现与服务监听。
- 协议回滚：保留本地安装路径，局域网传输失败不影响本机原有功能。
- 数据回滚：白名单与配对缓存异常时可一键清空并重新配对。
