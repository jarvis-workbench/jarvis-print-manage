# 踩坑与防坑手册

## 1. PowerShell 正则转义导致脚本报错
- 现象：`USBPRINT\` 结尾反斜杠在 `-match` 中成为非法正则，worker 持续报错。
- 防坑：
  - 优先用 `-like 'USBPRINT\\*'` 这类通配而不是复杂正则。
  - 对包含 `\` 的匹配，必须先做最小样例验证。

## 2. PowerShell 返回字段大小写不一致
- 现象：脚本输出 `Name/WorkOffline`，JS 代码按 `name/workOffline` 读取，导致状态判断失效。
- 防坑：
  - 读取脚本数据时统一 `pick(['name','Name'])` 兼容。
  - 关键字段（`PrinterStatus`、`WorkOffline`、`QueueStatus`）必须双写兼容。

## 3. USB 打印机场景误判就绪
- 现象：USB 已断开，列表仍显示就绪。
- 防坑：
  - 状态判断要组合：`workOffline`、`PrinterStatus`、`QueueStatus`、WMI 状态、PnP 在线信息。
  - 不要仅靠端口是否存在判断在线。

## 4. 虚拟打印机回流
- 现象：前端合并多源数据时把虚拟打印机重新显示出来。
- 防坑：
  - 在“最终聚合列表”层统一过滤虚拟打印机，不只在某一路源头过滤。

## 5. 多套列表导致数据源混乱
- 现象：同页面维护多套列表，无法确定展示依据，容易出现状态不同步。
- 防坑：
  - 仅保留单一聚合对象（当前为 `PrinterServerManage`）。
  - 页面仅消费聚合对象，不直连中间列表。

## 6. 编码问题（文档乱码）
- 现象：部分中文文档出现乱码（历史文件中存在）。
- 防坑：
  - 统一用 UTF-8 保存。
  - 提交前抽查关键文档可读性。

## 7. LAN 安装 USB 驱动误建端口
- 现象：网络驱动安装链路中，USB 档案在部分场景会误走 TCP 端口创建逻辑。
- 防坑：
  - USB 档案判定不能只看端口名，需同时结合 `usbVid/usbPid/usbVidPid/deviceSerial`。
  - 一旦判定为 USB 档案，安装脚本参数必须清空 `PREFERRED_PORT_HOST`，保持“仅安装驱动，不创建端口”。
