# 开发注意事项

## 状态管理约束
- 打印机管理使用单一聚合对象：`runtime.PrinterServerManage`。
- 禁止重新引入并行列表（如 `installedPrinters`、`driverIndexEntries`、`printerList`、`printerRuntimeState`）作为页面主数据源。
- 备份目录统一使用 `runtime.settings.backupDir`，不要新增重复字段。

## Electron 与脚本约束
- 与打印机/端口/驱动相关的系统调用必须经过主进程 IPC，不要在渲染进程直接做系统命令。
- PowerShell 脚本优先放在 `electron/config/script/ps/`，通过 `loadPsScript` 加载。
- 修改 `*.scps1` 后，要同步更新 `sign.json`，否则签名校验会失败。

## 设置持久化约束
- 系统设置文件路径：`electron/resource/config/system.json`。
- 打包后实际路径：`安装目录/resources/config/system.json`。
- 该文件为可变配置，不能放入 `app.asar`。

## UI 与交互约束（打印机管理）
- 安装向导是自定义面板，不使用 `el-dialog`。
- 输入组件统一使用 Element Plus 输入框。
- 打印机状态显示应贴近“打印机”信息，不做分散冗余展示。

## 代码修改约束
- 优先小步可验证改动，改完至少做语法检查（`node --check` / Vue SFC parse）。
- 不要在未确认情况下删除已有用户行为逻辑。
- 涉及打印机状态判断时，必须考虑中英文状态文本与状态码兼容。
