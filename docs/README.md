# 项目开发记忆库（docs）

更新时间：2026-03-28

这个目录是本项目的长期开发记忆库，目标是：
- 给当前维护者看，快速理解项目全貌。
- 给未来接手的人看，快速恢复上下文。
- 给未来对话中的模型看，减少“失忆”和重复踩坑。

## 阅读顺序
- `docs/01-项目简介.md`
- `docs/02-项目结构说明.md`
- `docs/03-开发注意事项.md`
- `docs/04-踩坑与防坑手册.md`
- `docs/05-功能与现状.md`
- `docs/06-协作约定.md`
- `docs/07-开发与升级日志.md`
- `docs/08-总结.md`
- `docs/09-文件级职责清单.md`
- `docs/10-命名规范与规划制度.md`
- `docs/planning/`（规划文档目录）

## 使用规则
- 每次做完功能、修复问题、调整架构后，必须补充/更新对应文档。
- 新增目录或关键文件时，必须同步更新 `02-项目结构说明.md`。
- 修复线上/测试问题后，必须同步更新 `04-踩坑与防坑手册.md`。
- 与用户协作规则变化后，必须同步更新 `06-协作约定.md`。
- 用户提出“规划”时，必须先在 `docs/planning/` 落盘方案并待确认后再编码。
